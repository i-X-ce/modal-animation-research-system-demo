# ViewTransition Demo

## 概要 (Overview)

本ディレクトリは、焼肉屋のセルフオーダーシステムを模した UI を通じて、**モーダル表示時のアニメーション（特に View Transitions API を用いたカード拡大アニメーション）の UX 的な効果を定量的に測定・実験する**ためのテストページです。

ユーザーが商品（肉）のカードをタップし、詳細モーダルから注文カートへ追加する一連のフローを実装します。

以下の要件を満たした上で、できるだけ洗練されたデザインを心がけて制作してください。

## 技術スタック (Tech Stack)

- **UI Framework:** Material-UI (MUI) ※独自スタイリングは極力避け、MUI のテーマとコンポーネントシステムを最優先で使用すること。
- **State Management:** Zustand (グローバルステート管理)
- **Animation:** View Transitions API, CSS Transitions (比較用)

## 必須要件と機能仕様 (Features & Requirements)

### 1. アニメーション設定機能 (Animation Controls)

画面上のコントロールパネルから、以下のパラメータを動的に変更できること。

- **アニメーションの種類:**
  - View Transitions (カード拡大モーダル)
  - 従来のアニメーション (Fade In / Slide Down)
  - アニメーションなし (None)
- **パラメータの調整:**
  - アニメーション時間 (Duration)
  - イージング (Easing)
  - モーダルの画面占有率 (Size/Coverage)

### 2. 焼肉セルフオーダー UI (Yakiniku Order UI)

- **メニュー一覧画面:**
  - 商品カードがグリッド状に並ぶ。
  - 各カードには「商品写真」「部位名」「価格」を必ず含める。
- **詳細モーダル:**
  - カードタップで開く。
  - 部位の詳細説明、価格、注文個数の設定（+, - ボタン）、カート追加ボタンを配置。
- **注文（カート）ページ/ドロワー:**
  - 現在カートに入っている商品と合計金額を表示し、注文確定アクションが行えること。

## アーキテクチャと実装ルール (Architecture & Rules)

### State Management (Zustand)

状態管理は Zustand を用い、以下の Store に分割（または統合）して管理すること。

1. `useModalStore`: モーダルの開閉状態、選択中の商品データ、現在適用されているアニメーション設定の保持。
2. `useCartStore`: カート内の商品リスト、個数、合計金額の計算ロジックの保持。
   ※ Store のデータ構造（特に商品アイテムや注文内容）は、Zod 等を用いて型定義を明確にし、Single Source of Truth (SSOT) を維持すること。

### Layout & Modal Root

モーダルコンポーネントは各カード内に個別で配置するのではなく、**対象ページの `layout.tsx` (またはルートに近い場所) に配置**し、`useModalStore` の状態を監視してグローバルに開閉を制御すること。

### 依存関係とインポート (Imports)

- **兄弟ディレクトリからのインポート禁止:** 当該ページのディレクトリ内（`./`）で自己完結させるか、プロジェクトルートの共通コンポーネント（`@/components`）のみを使用すること。
- **MUI の徹底活用:** アニメーションテストという性質上、レイアウトやボタン等の基本 UI は MUI の `Box`, `Stack`, `Card`, `Dialog` (従来型用) 等を利用して構築すること。
